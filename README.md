# 减肥计划工作台

一个黑白科技感的减脂记录工作台 Demo。核心逻辑是按天录入饮食、训练、睡眠、步数、热量和体重，再由周、月、半年页面自动汇总成规划与复盘视角。

## 在线访问

启用 GitHub Pages 后，项目可以通过以下地址访问：

https://7hu0v0.github.io/jianfeiworkspace/index.html

如果页面暂时 404，请在仓库 `Settings -> Pages` 中选择：

- Source: `Deploy from a branch`
- Branch: `main`
- Folder: `/ (root)`

保存后等待 GitHub Pages 发布完成即可。

## 当前功能

- 左侧时间侧边栏，包含日录入、周规划/复盘、月规划/复盘、半年规划/复盘
- 日录入是核心数据源：每天填写热量、饮食、训练、体重、睡眠和步数
- 周、月、半年页面从每日记录自动汇总，不再和日录入共享同一份表单状态
- 直接填写热量配额、今日摄入、运动消耗目标、周期起始体重和今日体重
- 自动计算热量缺口、减重数值、目标完成度和综合推进分
- 饮食文本输入，根据常见食材关键词估算蛋白质
- 步数、睡眠、训练分钟数输入与目标完成展示
- 训练内容记录与训练进度条
- 管理员模式：公开访问默认只读，输入管理员密码后解锁日录入编辑
- 纯静态实现，无需后端服务

## 本地预览

直接用浏览器打开 `index.html` 即可。

也可以启动一个本地静态服务：

```bash
python3 -m http.server 4173
```

然后访问：

```text
http://localhost:4173/index.html
```

## 说明

当前版本是一个前端 Demo。日录入数据暂存在页面内存中，刷新后会回到 demo 初始数据；后续可接入浏览器本地存储或后端数据库。

蛋白质估算基于本地关键词规则，例如鸡胸肉、鸡蛋、三文鱼、酸奶、乳清蛋白等，只用于产品原型展示，不等同于精确营养数据库。

训练建议也是基于输入目标和完成率生成的轻量提示。后续可以接入更完整的食物营养数据库、训练模板和用户历史记录。

管理员模式是前端轻量门禁，适合防止公开页面被随手误改；它不是后端鉴权，不适合保护敏感数据。
